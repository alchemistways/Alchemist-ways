# Alchemist Ways — Phase 1 Brief

## Goal

Build trust around the book and vision, and convert visitors into waitlist email leads. A 1-on-1 call is a secondary action only.

## CTA hierarchy

1. **Primary:** Join the Waitlist (email only)
2. **Secondary:** Book a 1-on-1 Call (final CTA section only, muted link — not equal to waitlist in hero)

## Tone

Calm, intentional, premium. Transformational without dark mysticism or dense “portal” UI. White-space first; gold and orange carry structure.

## Phase 1 — in scope

- Single landing page (`index.html`) on GitHub Pages
- Nav: Logo · Book · Vision · Community · Join Waitlist
- Sections: Hero → Book/Vision → Circular journey map → Why Alchemist Ways → Final waitlist + secondary call
- Real form endpoint wiring (Formspree-style until client tool is confirmed)
- Mobile-first polish
- Markdown docs for brand, scope, and workflow

## Phase 1 — out of scope

Digital products, high-ticket funnels, multi-page site, CRM segmentation, community/app, paid ads, new frameworks or dependencies unless explicitly requested.

## Success criteria

- Visitor understands Alchemist Ways + the book in the first viewport
- White-space canvas with blended gold/orange structure (not a dark ink portal)
- One dominant waitlist path; call clearly secondary
- Form submissions leave the browser (or clear configure-endpoint state)
- Map reads as a circular journey on desktop; stacks cleanly on mobile
- Docs exist so future work does not re-litigate Phase 1 scope
